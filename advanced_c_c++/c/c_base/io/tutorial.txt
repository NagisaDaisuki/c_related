size_t fread ( void *ptr, size_t size, size_t count, FILE *stream );
size_t fwrite( const void *ptr, size_t size, size_t count, FILE *stream );

参数大解剖（记住这个口诀：“地址、大小、数量、文件”）：

    ptr（内存地址）： 数据从哪里来（写），或者读出来放到哪里去（读）。

    size（单块大小）： 每个数据块有几个字节？（通常用 sizeof(你的类型)）。

    count（数量）： 你要搬运多少个这样的块？

    stream（文件指针）： 目标文件。

fwrite 是把数据写入二进制文件
fread 是把二进制文件的数据抠出来


💣 极客避坑指南：C 语言经典的致命陷阱

如果你在面试时被问到 fwrite，面试官大概率会拿下面这三个坑来考你：
陷阱一：返回值到底是什么？

fread 和 fwrite 返回的是**“成功操作的块数（count）”**，绝对不是字节数！
如果 fread(ptr, 4, 10, file) 返回了 8，意思是它成功读了 8 个块（也就是 32 字节）。如果文件到底了，或者出错了，返回值会小于你要求读的数量（10）。

陷阱二：不要跨平台读取（大端小端危机）

如果你在你的 Intel 电脑（小端序，Little-Endian）上用 fwrite 存了一个 int，然后把这个文件发给了一台古老的路由器（大端序，Big-Endian）用 fread 读取。
因为它们是二进制“原样复印”，大端机器读出来的数据会是完全颠倒的，血量 150 可能会变成几千万！
（文本文件不存在这个问题，因为字符 150 大家都认识）。

陷阱三：极其恐怖的“浅拷贝”指针炸弹（终极 Bug）

typedef struct {
    int id;
    char *name; // 注意！这是一个指针！
} BadPlayer;

如果你把这个结构体用 fwrite 写进文件，你存进去的只是一个内存地址（比如 0x7fffc001），而不是真正的字符串名字！
等你下次用 fread 读出来时，那个内存地址早就失效了（可能被分配给了别的程序）。只要你一访问 name，程序立刻当场崩溃（Segfault）！

记住： 用 fread/fwrite 操作的结构体，里面的字符串必须是定长数组（如 char name[50];），绝对不能是指针！
